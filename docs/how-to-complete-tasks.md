# Hot To Complete Tasks

## Где искать задачи

Задачи публикуются в Issues репозитория. Надо брать только те задачи,
которые создал автор репозитория (https://github.com/nelkor).

## Ветка

Для новой задачи актуализируем ветку `main` и создаём от неё новую.
Имя ветки начинаем с `feature/` или `bug/` в зависимости от сути задачи.
Добавляем номер Issue и в `kebab-case` краткую суть задачи.
Пример: `feature/90-connect-font`.

## Коммиты

Одна задача — один коммит. Имя коммита — "subj".

## Изменения кода

В основном, изменения кода должны касаться только текущей задачи.
Если для решения задачи будет оптимальным поправить стороннюю логику,
это также надо сделать.

Если нужно импортировать что-то из другого слайса, но он это не ре-экспортирует,
можно добавить ре-экспорт в `index.ts` того слайса.

Добавлять новые npm-библиотеки "просто так" нежелательно,
но если есть причина, то можно.

## Pull Request

Результатом выполнения задачи является Pull Request к ветке `main`.
Имя PR начинается с emoji `:sparkles:` для `feature` и `:bug:` для `bug`.
Далее в квадратных скобках номер Issue и в `Title Case` имя задачи.
Пример: `✨ [90] Connect Font`.

В описании PR следует указать:
- Сколько времени заняло выполнение задачи.
- Сколько и каких ресурсов было потрачено.
- Были ли моменты, не предвиденные условием задачи.
- Какие новые npm-библиотеки были добавлены.
- В чём заключается решение и почему оно такое.
